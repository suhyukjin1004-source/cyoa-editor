# CYOA 제작 툴

브라우저에서 바로 동작하는 **인터랙티브 CYOA(Choose Your Own Adventure) 제작 도구**입니다.
설치·빌드 과정이 없고(순수 HTML/CSS/JS), 완성한 작품은 `project.json` 으로 저장해
**Neocities** 같은 정적 호스팅에 그대로 올릴 수 있습니다.

두 가지 장르를 한 툴에서 모두 만들 수 있습니다.

- **포인트-빌드형** — 이미지·선택지 “행(row)”과 **포인트/통화 예산**으로 캐릭터·빌드를 구성 (r/makeyourchoice 스타일)
- **분기 서사형** — 장면(텍스트)을 읽고 선택지로 다음 장면으로 이동 (Twine 스타일)
- 두 형식을 한 작품 안에서 **섞을 수도** 있습니다.

---

## 빠른 시작

이 도구는 파일을 서로 불러오므로 **로컬 웹서버**에서 여는 것을 권장합니다.

```bash
cd <이 폴더>
python3 -m http.server 8765
```

브라우저에서 열기:

- 에디터: <http://localhost:8765/editor.html>
- 뷰어(미리보기): <http://localhost:8765/viewer.html>

> 처음 열면 데모 프로젝트(`project.json`, 하이브리드 예시)가 로드됩니다.
> 예제: `examples/sample-build.json`(빌드형), `examples/sample-story.json`(서사형).
> 뷰어에서 특정 파일을 보려면 `viewer.html?p=examples/sample-build.json` 처럼 엽니다.

## 화면 구성 (에디터)

| 영역 | 설명 |
|---|---|
| **상단바** | 새로 · 불러오기 · 저장(JSON) · 설정 · 🕸 연결망 · neocities 내보내기 · 미리보기 |
| **왼쪽 트리** | 오프닝 / 페이지 → (행 → 선택지) / 이동 링크 구조. `↑ ↓ 🗑` 로 순서변경·삭제, `＋` 로 추가 |
| **가운데 캔버스** | 실제 모습대로 표시. 요소를 클릭하면 오른쪽에서 편집 |
| **오른쪽 인스펙터** | 선택한 요소(페이지/행/선택지/링크)의 속성 편집 |

## 핵심 개념

- **오프닝(Opening)** — 플레이어가 로딩 후 처음 보는 시작 화면입니다. 제목·부제목·본문·시작 버튼·이미지를 편집할 수 있고, 이미지는 카드 안 또는 배경으로 표시할 수 있습니다. 카드 위치와 컴포넌트 간격은 캔버스에서 드래그하거나 인스펙터 숫자 입력으로 조정할 수 있으며, 추천 정렬 프리셋도 적용할 수 있습니다.
  - **자유 배치(캔버스)**: 인스펙터 「배치 → 자유 배치」를 켜면 **각 요소(이미지·제목·부제목·본문·버튼)를 캔버스에서 자유롭게 드래그**할 수 있습니다(Canva식). 드래그 중 다른 요소의 가장자리/중심이나 화면 중앙에 맞으면 **분홍 가이드선**이 뜨고 **스냅**됩니다. 좁은 화면(모바일)에서는 자동으로 가운데 세로 정렬됩니다. 「↕ 다시 세로 정렬」로 초기 배치로 되돌립니다.
- **페이지(Page)** — 작품의 한 화면. 유형이 둘 중 하나입니다.
  - `서사` : 본문 텍스트(HTML 일부 허용) — **본문 아래에 선택지 행을 둘 수도** 있습니다.
  - `빌드` : 선택지 행들의 모음
  - 두 유형 **모두 행(선택지 묶음)을 추가**할 수 있습니다. 트리에서 페이지의 **＋ 행 추가**를 누르세요(서사 페이지는 본문 다음에 선택지가 붙습니다).
- **행(Row)** — 페이지 안의 선택지 묶음(예: “종족”, “능력”). 선택 방식이 있습니다.
  - `단일(라디오)` : 행에서 하나만 / `다중` : 여러 개(최소·최대 지정 가능)
  - **랜덤 선택(🎲)**: 행 인스펙터 「랜덤 선택 (주사위)」을 켜면 플레이 화면에 🎲 버튼이 생깁니다. 요구조건을 통과하고 예산이 감당되는 선택지 중 하나를 룰렛 연출과 함께 무작위로 고릅니다(단일 행은 교체, 다중 행은 남은 슬롯에 추가). “출신을 굴려라” 같은 연출에 쓰세요.
- **선택지(Choice)** — 카드 하나. 제목·설명·이미지·**점수**·**요구조건**을 가집니다.
  - **여러 번 선택(다중 카운트)**: 인스펙터 「여러 번 선택」을 켜면 플레이 시 **− N +** 스테퍼로 같은 선택지를 여러 번 고를 수 있고, 점수는 **개수만큼 곱**해집니다(최소/최대 지정). 결과 이미지엔 `×N`으로 표시.
  - **자동 활성/해제**: 인스펙터 「자동 활성/해제」에서, 이 선택지를 고르면 다른 선택지를 **자동으로 선택/해제**하도록 연결(세트 구매 등).
  - **애드온(조건부 추가 텍스트)**: 「애드온」에서, 이 선택지를 고르거나 **표시 조건**을 만족하면 카드 아래에 추가 텍스트 블록이 펼쳐집니다.
- **통화(Currency)** — 포인트·체력·금화 등. 시작값과 색을 정의(설정 모달). 선택지의 점수로 증감합니다.
  - 점수: **음수 = 비용**, **양수 = 획득**. 예산이 음수가 되면(허용 안 한 통화) 선택이 막힙니다.
  - **조건부 점수(할인/조건부 비용)**: 점수 항목의 「＋ 조건」을 켜면, **그 조건을 만족할 때만** 해당 점수가 적용됩니다. 예: 갑옷을 기본 −50으로 두고 “전사 선택 시 +20” 항목을 더하면 전사일 때만 실질 −30(20 할인). 재생 중엔 **조건 충족한 항목만** 표시되고, 편집 화면에선 「조건부」 뱃지로 보입니다. (조건이 통화 값을 참조하면 **할인 전 값** 기준으로 판정됩니다.)
- **변수(Variable)** — 통화와 달리 **예산 잠금이 없는 자유 상태**입니다(설정 모달 「변수」에서 정의). 타입은 **수치** 또는 **참/거짓(깃발)**. 깃발·평판·관계·진행 플래그처럼 통화로 표현하기 어려운 상태에 사용.
  - **선택지 효과**: 선택지를 고르면 변수를 더하기/빼기/켜기 — **선택을 해제하면 되돌아갑니다**(되돌릴 수 있음).
  - **이동 링크 효과**: 링크로 페이지를 옮길 때 변수를 더하기/빼기/켜기/끄기 — **이동 1회만** 적용(이야기 진행 상태).
  - 요구조건 「변수 값」으로 잠금/숨김을 걸고, 본문에서 `{{var:변수id}}`로 값을 표시할 수 있습니다.
- **그룹(Group)** — 여러 행에 흩어진 선택지를 하나로 묶는 태그입니다(설정 모달 「그룹」에서 정의, 선택지 인스펙터 「그룹」에서 태그). 요구조건 **「그룹 선택 수」**(예: “능력 그룹에서 2개 이상”)와 🎒 **백팩 분류**에 쓰입니다.
- **글로벌 요구조건** — 자주 쓰는 조건 묶음을 **세트로 저장**해 여러 곳에서 재사용합니다(설정 모달 「글로벌 요구조건」). 요구조건에서 「조건 세트(글로벌)」로 참조하고, 본문에서는 `{{if:global:세트id}}`로 씁니다. 세트끼리 참조할 수도 있으며 순환 참조는 무시됩니다.
- **동적 텍스트** — 본문·설명·애드온에 토큰을 넣으면 선택에 따라 실시간으로 바뀝니다.
  - `{{cur:통화id}}` → 현재 통화 합계(예: `남은 포인트: {{cur:pt}}`).
  - `{{var:변수id}}` → 현재 **변수** 값(깃발은 예/아니오).
  - `{{word:단어id}}` → **동적 단어**(설정 모달 「동적 단어」에서 정의). 규칙 조건을 만족하면 그 값, 아니면 기본값.
  - `{{if:조건}}…{{else}}…{{/if}}` → **조건부 텍스트**. 조건: `choice:선택지id` · `var:변수id`(또는 `var:hp>=10`) · `cur:통화id>=50` · `group:그룹id>=2` · `global:세트id`, 앞에 `!`로 부정. 중첩 가능. (예: `{{if:var:brave}}용감하게 나섰다.{{else}}머뭇거렸다.{{/if}}`)
- **요구조건(Requirement)** — 배열의 **모든 조건을 만족**해야 선택/이동 가능 (AND).
  - `선택지` 조건: 특정 선택지를 선택함/선택안함
  - **`이 중 하나(OR)`**: 나열한 선택지 중 **하나라도 선택**(또는 **모두 미선택**)이면 통과. AND 목록 안에서 OR 그룹을 만들 수 있습니다.
  - `통화 값` 조건: 통화 값이 `>= <= > < == !=` 기준 충족
  - **`통화 비교`**: 두 통화를 서로 비교(예: 공격력 > 방어력)
  - **`그룹 선택 수`**: 그룹에 태그된 선택지 중 선택된 개수를 비교(그룹 정의 시 표시)
  - **`조건 세트(글로벌)`**: 저장해 둔 글로벌 요구조건 세트를 통째로 참조(세트 정의 시 표시)
  - 조건 미충족 선택지는 기본적으로 흐리게 표시(전역 설정에서 숨김으로 전환 가능).
  - **선택지별 숨김**: 선택지 인스펙터의 **「조건 미충족 시 숨김」** 체크 시, 그 선택지는 **요구조건을 만족하기 전까지 보이지 않습니다**(조건을 만족하면 나타남). “숨겨진 보상/분기”를 만들 때 사용. 편집 화면에서는 🙈 표식과 함께 항상 보입니다.
- **이동 링크(Link)** — 다른 페이지로 가는 버튼. 서사 분기와 하이브리드 전환에 사용. 요구조건/점수도 줄 수 있습니다.
  - **링크 숨김**: 이동 링크 인스펙터의 **「조건 미충족 시 숨김」** 체크 시, 요구조건을 만족하기 전까지 그 **버튼이 보이지 않습니다**(만족하면 나타남). “숨겨진 분기/통로”를 만들 때 사용. 편집 화면에서는 🙈 표식과 함께 보입니다.
- **순서 변경** — 트리의 `⋮⋮` 핸들로 페이지·행·선택지·이동 링크를 드래그해 순서를 바꿀 수 있습니다. 캔버스에서도 행·선택지·이동 링크의 `⋮⋮` 핸들을 드래그해 같은 부모 안에서 재정렬할 수 있습니다.
- **개인화** — 설정 모달의 **「개인화」** 에서 전체 레이아웃, 선택지 형태, 페이지 전환 효과와 속도를 고를 수 있습니다. 선택한 값은 공유 뷰어와 내보내기에도 반영됩니다.
- **배경 음악(BGM)** — 페이지 인스펙터의 **「배경 음악(BGM)」** 에서 오디오를 지정합니다.
  - **오디오 파일 업로드**(프로젝트에 내장, 작은 파일 권장) 또는 **URL/경로**(예: `audio/theme.mp3`, neocities `audio/` 폴더에 올린 파일) 둘 다 지원.
  - 음악을 지정한 페이지에 들어가면 그 곡이 재생되고, **음악이 없는 페이지는 이전 곡을 이어서** 재생합니다. 특정 페이지에서 멈추려면 **「이 페이지에서 음악 정지」** 를 체크하세요.
  - **스크롤형** 작품은 첫 번째로 음악이 지정된 페이지의 곡이 전체 배경으로 깔립니다.
  - 뷰어 상단의 **🔊/🔇** 버튼으로 켜고 끌 수 있습니다. (브라우저 정책상 소리는 **첫 클릭/입력 후** 시작될 수 있습니다.)
  - 인스펙터의 **▶ 미리듣기** 로 에디터에서 바로 들어볼 수 있고, ▶ 미리보기에서도 BGM이 재생됩니다.
- **이미지 편집(업로드 시)** — 이미지를 올리면(또는 기존 이미지의 **✎ 편집**) **편집 창**이 열립니다.
  - **크롭(자르기)**: 모서리 핸들로 영역 지정. **종횡비** 자유/1:1/4:3/3:4/16:9/원본 고정.
  - **회전·반전**: 90° 회전(↺↻), 좌우/상하 반전.
  - **리사이즈·압축**: 출력 **최대 너비**·**형식(WebP/JPEG/PNG)**·**품질** 조절. **예상 용량(KB)** 을 실시간 표시 → **project.json 용량을 크게 줄일 수 있습니다.** 기본값(최대 1600px·품질 0.85)만으로도 큰 원본이 자동으로 작아집니다.
  - 형식 참고: WebP는 작고 투명 지원(Chrome·Firefox). **Safari는 WebP 인코딩을 지원하지 않아 자동으로 PNG로 저장**됩니다. 투명이 필요 없으면 JPEG가 가장 작습니다.
- **이미지 배치(Layout)** — 페이지·행·**선택지** 인스펙터에 **이미지 배치** 항목이 있습니다. 페이지·행은 캔버스에서 이미지 위치/크기와 간격 핸들을 직접 드래그할 수도 있습니다.
  - `위치`: 텍스트 기준 **위 / 아래 / 왼쪽 / 오른쪽** (왼쪽·오른쪽이면 이미지와 글이 나란히 배치)
  - `너비 %`: 이미지 크기(가로). 좌/우 배치에서는 옆 칸의 너비. (선택지 카드는 좌/우 배치에서 적용)
  - `높이 px`: 페이지·행은 `0`=원본 비율, 값=그 높이로 잘라 표시. 선택지 카드는 `0`=표준 높이, 값=지정 높이.
  - 페이지·행 전용: `이미지-텍스트 간격`, `제목-콘텐츠 간격`, `콘텐츠-선택지 간격`, `이미지 정렬`, `텍스트 상자 폭/위치`, `텍스트 정렬`을 숫자·버튼·캔버스 핸들로 조정합니다.
  - 캔버스 드래그: 이미지/텍스트 상자의 가로 정렬 핸들을 움직이면 다른 컴포넌트와 캔버스 중앙에 맞춰 정렬 가이드가 표시되고 스냅됩니다.
  - 줄바꿈은 본문에서 **엔터=줄바꿈, 빈 줄=문단 나눔** 으로 됩니다(HTML도 허용).

## 작업 흐름

1. 왼쪽 **오프닝**에서 시작 화면의 제목·본문·이미지·버튼 문구를 정합니다.
2. **설정**(⚙)에서 제목·통화·테마·진행방식(페이지형/스크롤형)을 정합니다.
3. 왼쪽에서 **페이지**를 추가하고 유형(서사/빌드)을 고릅니다.
4. 빌드 페이지면 **행 → 선택지**를 추가하고, 각 선택지에 **점수·요구조건**을 설정합니다.
5. 서사/하이브리드면 **이동 링크**로 페이지를 연결합니다.
6. 상단 **▶ 미리보기**로 오른쪽 플레이어 시뮬레이션 패널을 열어 실제처럼 테스트합니다. 패널에서 **현재/모바일/태블릿/PC/사용자** viewport를 바꿔 화면 폭별 레이아웃도 확인할 수 있습니다.
7. **저장(JSON)** 으로 작업본을 보관하고, **neocities 내보내기**로 배포본을 만듭니다.

> **자동 저장**: 작업은 브라우저(localStorage)에 자동 저장되어 다시 열면 복원됩니다.
> 다른 컴퓨터로 옮기거나 백업하려면 반드시 **저장(JSON)** 으로 파일을 내려받으세요.

## 연결망(그물망) 뷰 🕸

상단 **🕸 연결망** 버튼을 누르면, 현재 작업 중인 CYOA의 **선택지·페이지가 서로 어떻게 연결되는지**를
힘-기반(force-directed) 그래프로 한눈에 볼 수 있습니다.

표시되는 연결:

| 선 | 의미 |
|---|---|
| 강조색 실선 화살표 | **요구(선택함)** — A를 선택해야 B가 열림 (선택지→선택지) |
| 빨강 점선 화살표 | **요구(선택안함)** — A를 선택하면 B가 막힘 |
| 강조색 점선 화살표 | **이동 게이트** — 특정 선택지가 있어야 갈 수 있는 페이지 이동 |
| 파랑 실선 화살표 | **페이지 이동** — 이동 링크(서사 분기) |
| 옅은 선 | **포함** — 선택지가 어느 페이지에 속하는지 |

조작: **노드 드래그**로 이동 · 빈 곳 드래그로 **팬** · **휠**로 확대/축소 · **맞춤** 버튼으로 화면에 맞춤 ·
**재배치**로 레이아웃 다시 계산 · 노드에 **마우스를 올리면** 연결된 것만 강조 · 노드 **더블클릭**으로 그 요소를 편집 패널에서 바로 엽니다.

> 요구조건을 복잡하게 엮을수록 “어떤 선택이 무엇을 잠그고 여는지” 추적이 어려워지는데, 이 뷰로 전체 의존 관계를 시각적으로 검토할 수 있습니다.

## 플레이어 편의 기능 (뷰어) 🎒

뷰어(그리고 에디터 ▶ 미리보기) 상단바에서 쓸 수 있습니다.

- **🎒 백팩** — 지금까지 고른 선택지를 **실시간 사이드 패널**로 봅니다. 그룹이 정의돼 있으면 그룹별, 아니면 행별로 분류되고, 각 항목의 ✕로 패널에서 바로 해제할 수 있습니다(자동 활성/연쇄 해제 규칙 그대로 적용).
- **💾 저장 슬롯** — 백팩 패널 하단에서 현재 상태를 **이름 붙여 저장**(작품별 최대 10개)하고 언제든 불러옵니다. 진행 자체는 원래대로 자동 저장되며, 슬롯은 “다른 빌드 비교” 같은 용도의 수동 저장입니다. 저장 형식은 빌드코드와 같아 서로 호환됩니다.
- **🌓 밝기 전환** — 플레이어가 제작자 테마의 명도를 반전한 밝은/어두운 변형으로 전환할 수 있습니다(선호는 기억됨). 테마가 연출의 일부인 작품은 설정(⚙) 「플레이어 밝기 전환(🌓) 허용」을 꺼서 감출 수 있습니다.

## 결과 이미지 추출 🖼

뷰어(그리고 에디터 ▶ 미리보기) 상단의 **🖼 결과 이미지** 버튼을 누르면, **플레이어가 선택한 선택지들**을
한 장의 이미지(결과 시트)로 저장합니다. 제목·통화 합계·행별로 고른 선택지(썸네일+제목+점수)가 정리됩니다.

- 형식: **WebP** (Chrome·Firefox 등 지원 브라우저). **Safari는 WebP 인코딩을 지원하지 않아 자동으로 PNG로 저장**됩니다(그림 내용은 동일).
- 이미지는 캔버스로 직접 그려 외부 라이브러리가 필요 없습니다. 업로드한 이미지(데이터 URL)는 정상 포함됩니다.
- 참고: 교차 출처(CORS 미허용) 외부 이미지 경로는 보안상 캔버스에 못 그릴 수 있어, 그 경우 해당 썸네일만 빠집니다. (에디터로 올린 이미지는 영향 없음)

## 내보내기 (Neocities용)

상단 **neocities 내보내기**에 4가지 방법이 있습니다.

> ⚠ ①·② 방식은 엔진·뷰어 파일을 함께 묶기 위해 **로컬 웹서버로 열었을 때만** 동작합니다(빠른 시작의 `python3 -m http.server` 참고). `file://`로 직접 열었다면 안내 메시지가 표시됩니다.

1. **단일 파일 (`index.html`)** — 엔진·뷰어·스타일·데이터·이미지를 전부 한 파일에 인라인. Neocities에 이 파일 **하나만** 올리면 끝. *(가장 쉬움)*
2. **공유 뷰어 세트 (5개 파일)** — `index.html` · `viewer.js` · `engine.js` · `styles.css` · `project.json` 을 한 번에 내려받아 함께 업로드. 이미지·음악이 많을 때 권장(이미지/음악을 경로로 쓰면 `images/` · `audio/` 폴더도 함께).
3. **`project.json` 만** — 작업본 백업용.
4. **이미지 분리** — 이미지를 개별 파일로 내보내고 `project.json` 은 `images/...` 경로로 참조.

> 공유 뷰어는 첫 화면에 **제목·소개 + ▶ 시작하기** 시작 화면을 보여줍니다(설정 → 「공유 뷰어 시작 화면 표시」로 끌 수 있음). 시작 버튼 클릭이 배경 음악 재생을 시작하는 첫 동작이 됩니다.

자세한 업로드 절차는 [DEPLOY-neocities.md](DEPLOY-neocities.md) 참고.

## 확장 (고급)

코드를 몰라도 쓸 수 있지만, **원하는 기능을 직접 구현**하고 싶다면 두 가지 확장 지점이 있습니다.

- **커스텀 CSS** (설정 → 테마) — 외형을 자유롭게 바꿉니다.
- **커스텀 JS** (설정 → **확장 — 커스텀 JS**) — 작품이 로드될 때 실행되는 스크립트로, `api.on("render", …)` 훅을 통해 렌더에 끼어들거나 요소를 추가할 수 있습니다. `CYOA` 엔진 API와 `project`·`state` 에 접근합니다.

둘 다 `project.json` 에 저장되어 **내보내기에 자동 포함**됩니다. 커스텀 JS는 안전을 위해 **내가 내보낸 작품에선 자동 실행**되지만, **남의 `project.json` 을 열 땐 실행 전 확인**을 거칩니다.

> 훅 목록·API 표·예제·보안 모델은 [EXTEND.md](EXTEND.md) 참고.

## 파일 구성

| 파일 | 역할 |
|---|---|
| `editor.html` / `editor.js` | 제작 툴(에디터) |
| `index.html` | **Neocities 공유용 진입점**(시작 화면 포함). `viewer.js`로 재생 |
| `viewer.html` | 로컬/내부용 뷰어 (index.html과 동일 구조) |
| `viewer.js` | 공유 뷰어 로직(시작 화면·상단바·BGM·결과 이미지 등) |
| `engine.js` | 공유 엔진(렌더·요구조건·통화·BGM 판정). 에디터·뷰어가 함께 사용 |
| `styles.css` | 테마·UI 스타일 |
| `project.json` | 뷰어가 기본 로드하는 작품 데이터 |
| `examples/` | 빌드형·서사형 예제 |
